HTML Intro

HTML is a text document. It gives structure to a webpage.

How People Access the Web

People use web browsers to access the web through a number of devices. The majority of those are on mobile phones or tablets. The lower percentage of access comes from desktop browsers on a laptop or pc.

When a user access a web page it makes a call to a web server. The webapge is hosted by web hosting company

How Websites are Created

All websites use HTML and CSS.

In addition to the HTML and CSS some websites uses images and Javascript to make the site more engaging and dynamic.

Larger sites may use a Content Management System or CMS. Even more complex sites such as e-commerce sites will use databases withy complext tables to make requests to and display the information on screen.

  1. Connect to the web via Internet Service Provider
  2. Computer contacts a Domain Name System (DNS) using your Internet Prodvider address
  3. The DNS returns a unique number that will allow your browser to contact the web server of the website you requested.
  4. The web server sends the page back to your browser.

Elements

Elements are the building blocks of an HTML document.

Element Breakdown

An element is made up an opening tag and a closing tag.

An opening tag has a less-than sign followed by a character (depending on the element needed). After the character comes a greater-than sign. A closing tag starts with a less-than sign followed by a forward slash. Then the same character used in the first tag followed by a greater-than sign.

Element Attributes

Different elements have attributes that add functionality to an element. The structure of the attribute is denoted by the attribute name followed by an = sign. Then, the value of the attribute is placed between quotation marks after the equal sign.

Structure

As stated before, elements make up the building blocks of an HTML document. The head, title and body elements are key to any HTML document.

Head, Title, Body

The head goes before the body and contains the title element. It includes metadata - descriptive elements.

The title comes inside the head element. It is what shows up on the browser tab.

The body element contains all of the markup that is displayed in the browser.

Intro to Tables

Tables are good for structuring data in an organized way.

Creating a Table

Table rows and columns are put in between the table tags.

Table Data

Table data uses td tags. Table data are the cells in a table. They are placed inside the table rows.

Table Headings

Table headings use the th tag. They are place in a row by themselves.

Table Borders

Tables can have borders by adding inline styling or in the styles sheet. Do not use the borderattribute as it is deprecated.

Spanning Columns

To allow for a td element to span multiple columns use the colspan attribute.

Spanning Rows

To allow for a td element to span multiple rows use the rowspan attribute.

Table rows use tr tags.

Table Body

The table body or tbody element encompasses the table rows and table data elements.

Table Head

The table head or thead element is the top most row in the table element.

Table Footer

The tfoot element is the bottom of the table.

Styling with CSS

Tables, by default, are very bland. They have no borders, the font color is black, and the typeface is the same type used for other HTML elements.

Just like an HTML document, we can add styles via the tag, class or id.

By using a video element, we can add videos to our website. The element makes it clear that the page is attempting to display a video to the user.

Some attributes can alter a video playback include:

The embed element which can embed any media content including videos, audio files, and gifs from an external source. This means that websites that have an embed button have some form of media content that can be added to other websites.

CSS

Intro to CSS

CSS allows us to create rules to specify how the content of an element should appear on the page. Different browsers interpret some CSS rules differently.

How CSS Works

The key to understanding how CSS works is to imagine that there is an ivisible box around every HTML element.

Block level elements look like they start on a new line. Examples are h1 to h6, p and div elements.

Inline elements flow within the text and do not start on a new line. Examples include b, i, img, em, span.

CSS allows you to create rules that control the way that each individual box(and the contents of that box) is presented.

Rules, Properties and Values

CSS works by associate rules with HTML elements. You use the selector from HTML, such as the element tag, class or id. The declaration is put inside curly braces as a key-value pair.

Whilst you can use styles internally in an HTML document, using an external CSS file is better practice.

Some selectors include:

SELECTOR MEANING EXAMPLE
UNIVERSAL SELECTOR Applies to all elements in the document * {}
Targets all elements on the page.
TYPE SELECTOR Matches element names h1, h2, h3 {}
Targets the h1, h2, h3 elements
CLASS SELECTOR Matches an element whose class attribute has a value that matches the one specified after the period (or full stop) symbol .note {}
Targets any element whose class attribute has a value of note
p.note {}
Targets only p elements whose class attribute has a value of note
ID SELECTOR Matches an element whose id attribute has a value that matches the one specified after the poiund or hash symbol #introduction {}
Targets the eelement whose id attribute has a value of introduction
CHILD SELECTOR Matches an element that is a direct child of another li>a {}
Targets any a elements that are children of an li element (but not other a elements in the page)
DESCENDANT SELECTOR Matches an element that is a descendant of another specified element (not just a direct child of that element) p a {}
Targets any a elements that sit inside the p element, even if there are other elements nested between them
ADJACENT SIBLING SELECTOR Matches an elementthat is the next sibling of another h1+p {}
Targets the first pelement after any h1 element (but not other pelements
GENERAL SIBLING SELECTOR Matches an element that is a sibling of another, although it does not have to be the directly precending element h1-p {}
If you had two pelements that are siblings of an h1 element, this rule would apply to both

CSS rules cascade based on three factors. 1. If two or more rules that apply the same element, the last one will tak precendent; 2. If one selector is more specific that the others, the more specific rule will take precendence over more general ones.; 3. using the !important to consider it more important that other rules.

Font-family or color properties on the body element are inherited properties and apply to most elements on the document. background-color or border properties are not inherited by child elements. You can force inheritence by using the inherit rule.
